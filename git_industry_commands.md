3)repositary status and inspection commands
1.git status

Command Name:
git status

Syntax:
git status

Purpose:
Shows the current state of the repository: staged, unstaged, and untracked files.

Example:
git status

Screenshot:![Uploading git status.png…]()


2.git log

Command Name:
git log

Syntax:
git log

Purpose:
Displays a detailed list of commits in the repository.

Example:
git log

Screenshot:![Uploading git log.png…]()

3. git log --oneline

Command Name:
git log --oneline

Syntax:
git log --oneline

Purpose:
Displays commits in a condensed single-line format.

Example:
git log --oneline

Screenshot:![Uploading git log oneline.png…]()

4.git log --graph

Command Name:
git log --graph

Syntax:
git log --graph --oneline --all

Purpose:
Shows a visual graph of the commit history.

Example:
git log --graph --oneline --all

Screenshot :![Uploading git log --graph --oneline --all.png…]()


5.git show

Command Name:
git show

Syntax:
git show <commit-hash>

Purpose:
Displays details of a specific commit (changes, author, date).

Example:
git show abc1234

Screenshot :![Uploading git show.png…]()


6.git diff

Command Name:
git diff

Syntax:
git diff

Purpose:
Shows unstaged changes between your working directory and the last commit.

Example:
git diff

Screenshot :![Uploading git diff.png…]()


7.git diff --staged

Command Name:
git diff --staged

Syntax:
git diff --staged

Purpose:
Shows changes that are staged and ready to commit.

Example:
git diff --staged

Screenshot:![Uploading git diff staged.png…]()

8.git blame

Command Name:
git blame

Syntax:
git blame <file-name>

Purpose:
Shows who last modified each line of a file.

Example:

git blame git_industry_commands.md

9.git reflog

Command Name:
git reflog

Syntax:
git reflog

Purpose:
Displays a log of all reference updates in your repository, including HEAD movements.

Example:
git reflog

Screenshot :![Uploading git reflog.png…]()

Command Name:
git shortlog

Syntax:

Purpose:
Shows a summary of commits grouped by author.
